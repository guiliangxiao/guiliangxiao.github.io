Ubuntu20.04安装坚果云的曲折之路

发表于 2020-03-25 更新于 2020-06-22 分类于 Ubuntu

因为Ubuntu20.04系统中装了Anaconda3，默认的python版本anaconda自带的Python3.7，坚果云安装和运行需要python2的环境，这篇文章记录了自己当时解决这个问题的过程。

记录一下

  • 我在安装的时候试了下这篇博文中==基于update-alternatives==的方法。首先把/usr/bin/python2.7和~/opt/anaconda3/bin/python3.7两个python版本添加到update-alternatives命令中去，至于ubuntu系统自带的Python3（位于/usr/bin/python3.8）添不添加我觉得关系不大。
  • 添加完成之后用命令update-alternatives --config python把系统默认的python版本切换到python2，然后奇怪的事情就是默认的python还是Python3，没换到我想要的python2版本，然后后面每次一打开终端都会出现一个ImportError:No module named conda.cli，运行conda也是未发现命令
  • 谷歌一下，上面的ImportError应该是python版本不对的问题导致的，我猜应该是被我切换到了Ubuntu系统自带的Python3（当时应该可以用软连接的方法把python切换到python2的版本，当时没注意，也没想到），最后无奈只好用sudo rm -r ~/opt/anaconda3把anaconda卸载了。然后系统默认的python版本就回到了python2，如果还是python3的话可以用软连接的方法sudo ln -s /usr/bin/python2.7 /usr/bin/python把系统默认的python版本换成了Python2.7。
  • 然后通过官方的方法先下载deb包，然后用sudo dpkg -i nautilus_nutstore_amd64.deb和sudo apt-get install -f命令安装之后还是启动不了坚果云，后面看了下，是sudo apt-get install -f命令没有成功安装依赖包，然后是依赖怎么也安装不成功，现在想一下应该是Ubuntu系统版本太新的原因
  • 然后参考这篇博文的方法，换种方法安装
  • 然后在运行~/.nutstore/dist/bin/nutstore-pydaemon.py语句时出现了一个错误，仔细看了下是运行的这个文件nutstore-pydaemon.py的第93行出现了一个返回值Focal不能转换成float的问题，这里的Focal是Ubuntu20.04版本号，估计还是版本兼容的问题，然后gedit ~/.nutstore/dist/bin/nutstore-pydaemon.py打开文件把代码改一下，找到出错的==第93行==把原来的return float(ver) >= 18.04直接改成return True。

  • 最后还是在这个文件里面把开头的python版本改成系统的python2版本，然后就可以愉快的使用anaconda3和坚果云了，不用担心默认版本不兼容的问题了，然后再把ananconda3重新安装上，具体ananconda3重新安装参考这篇文章
  • 后面我在装好anaconda3的时候我到目录/usr/bin里面用命令ls -all看了一下python指向的默认python版本竟然还是python2，但是我在终端里面运行python的默认版本却是python3，所以怀疑最先一开始通过改python默认版本的方法来安装坚果云是行不通的
